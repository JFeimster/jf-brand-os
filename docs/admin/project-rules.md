# Project Rules

## Purpose
Keep this repo clean, usable, and free of duplicate sludge.

## Rules
1. Source-of-truth files must be updated before derivative files.
2. Do not invent metrics, dates, titles, or achievements.
3. Archive deprecated drafts instead of overwriting useful history.
4. Prefer one canonical file per asset type.
5. Use markdown for docs unless there is a strong reason not to.
6. Prompts should reference source docs rather than re-explaining everything from scratch.
7. Website copy should stay aligned with `docs/brand-bio/` and `docs/media-kit/`.

## Naming conventions
- lowercase
- hyphen-separated filenames
- folder names grouped by purpose

## Tone guardrail
Direct, strategic, human, credible, a little edge, zero fake guru perfume.
