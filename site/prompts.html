Category

Prompt documentation for repeatable writing, build, and maintenance workflows.

This collection contains reusable prompt packs that keep future site changes, bio generation, and documentation work consistent with the source-of-truth layer instead of relying on memory, improvisation, or prompt roulette.

Why this category exists

To standardize how bios, pages, docs, and supporting assets get generated or revised so the repo scales cleanly instead of becoming thread-dependent chaos.

Best use cases

Bio generation, site-file generation, doc maintenance, structured revisions, category-page expansion, and future prompt packs for new collections.

Where to start

Use the prompt pack that matches the asset type, but always feed it the right source docs first. Prompts are multipliers, not substitutes for source-of-truth files.

Start with the right source docs, then use the matching prompt pack, then review the output against the category page and source-of-truth files before publishing. Prompts should accelerate discipline, not bypass it.
