Suggested paths

Different visitors need different entry points. These routes keep people from wandering into the wrong layer of the system.

Need a public-facing bio?

Start with the master identity, then pull from the platform bios file, then sanity-check against proof points.

Need resume or role framing?

Start with resume master, then use the identity and proof-point docs to sharpen positioning without inventing anything.

Need site or media copy?

Start with About, one-sheet, and ecosystem map files so the copy reflects the full system instead of one thin page.

Use the docs before you touch the public pages.

That one habit alone will prevent half the drift, duplication, and weird brand mutations that happen when people start from a blank page instead of the system.
