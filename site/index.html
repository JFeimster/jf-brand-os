Repo workflow

A founder-friendly flow for keeping this system organized instead of letting every new thread spray files all over the place.

  1. 1. Update source docs first
    Touch the canonical files before revising site copy, social bios, speaker sheets, or derivatives.
  2. 2. Pull from receipts before polishing
    Strengthen proof points and chronology before chasing pretty words.
  3. 3. Use prompt packs to generate downstream assets
    Let the repo drive the outputs, not random memory fragments from five different chats.
